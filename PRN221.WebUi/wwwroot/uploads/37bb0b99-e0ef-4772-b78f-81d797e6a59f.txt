1. MAVEN LÀ GÌ? HAY HƠN ANT HAY KHÔNG?
- Là 1 build tool - 1 công cụ - 1 chương trình giúp đóng gói dự án Java thành .jar .war
  Để có thể đóng gói được dự án Java thành .jar .war thì Maven yêu cầu dân dev phải tổ chức thư mục source code 
  .java theo cách Maven muốn 
  Build tool nào thì phải tổ chức thư mục source code theo cách tool đó muốn

2. MỘT SỐ INFO LIÊN QUAN ĐẾN MAVEN
- Profile - hồ sơ của 1 file.jar ~~~ profile MXH của ta, có nhiều info mô tả cho .jar
1 file jar bất kì luôn có 3 info sau:
	- groupId: tên tổ chức/cty và tên dự án sở hữu file.jar này
	ví dụ: com.microsoft.sqlserver -> viết theo tên miền đảo ngược
					  chính là 1 phần của package chứa .jar 
	- artifactId: tên của file jar, là 1 phần của dự án mà file .jar thuộc về 
			sqlserver-jdbc
	- version: phiên bản của file .jar
3 INFO NÀY ĐỂ NHẬN DIỆN DUY NHẤT 1 FILE .JAR TA SẼ ADD VÀO PROJECT CỦA TA 
>>>>> KHI KHAI BÁO, ADD DEPENDENCY VÀO TRONG FILE POM.XML, NHỚ LIỆT KÊ 3 INFO TRÊN, MAVEN SẼ BIẾT TÌM FILE .JAR
ĐÚNG ĐẮN VÀ TẢI VỀ CHO BẠN 
*MỘT DỰ ÁN THEO KIỂU MAVEN, DÙNG MAVEN ĐỂ BUILD
 MỘT THƯ VIỆN/DEPENDENCY KHI CHƠI VỚI MAVEN CẢ 2 ĐỀU PHẢI KHAI BÁO PROFILE GỒM 3 INFO NHƯ ĐÃ NÊU!!!

* MAVEN SẼ ĐI TÌM THƯ VIỆN Ở ĐÂU???
  Trên Internet có 1 website, 1 nhà kho chứa toàn bộ các file .JAR trên thế giới com
  nằm ở https://mvnrepository.com
  mỗi file .jar ở trên kho này đều có 1 profile gồm 3 info: groupId, artifactId, version
  có sẵn cú pháp khai báo .jar này, bạn chỉ việc copy cú pháp này về và dán vào file POM.XML là xong 
  -> add dependency sau vào pom.xml
	<dependency>
		<groupId></groupId>
		<artifactId></artifactId>
		<version></version>
	</dependency>
CÚ PHÁP ĐỂ ĐỊNH VỊ DUY NHẤT 1 FILE .JAR Ở KHO TỔNG TOÀN CẦU

3. CÁCH CHƠI VỚI MAVEN
- Tạo dự án kiểu Maven: nhớ khai báo profile cho dự án: groupId, artifactId, version
- Khai báo các dependency cần dùng, cú pháp lấy ở mvnrepository.com
  dán vào file pom.xml, ở section <dependencies>
- Khai báo các plugin cần dùng, cú pháp lấy ở mvnrepository.com
  dán vào file pom.xml, ở section <build> 
- Clean & Build để Maven tự tải file .jar tương ứng về máy mình/local
  tự add vào project, tự build app, tự chạy bộ Unit Test luôn, và đóng gói thì lệ thuộc vào mình yêu cầu
  Maven làm thế nào!!1
  
  Clean & Build của NetBeans mặc nhiên nó đi gọi ANT hoặc MAVEN tùy loại dự án
  Khi bạn cài NetBeans đồng nghĩa bạn cài luôn ANT và MAVEN (gói combo)
  Thực tế, bạn ko thèm xài MAVEN đi kèm NETBEANS, bạn cài bộ maven rời, có riêng lẻ maven trên mạng
  MAVEN, ANT, TOMCAT,... Là sản phẩm của org.apache
